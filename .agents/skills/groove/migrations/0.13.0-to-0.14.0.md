# Migration: 0.13.0 → 0.14.0

## Outcome

`.groove/index.md` frontmatter is restructured to the new config schema: `tasks` becomes a nested block with `backend`, `list_limit`, and `analyse_limit`; `recent_memory_days` moves to `memory.review_days`; `memory:` and `specs:` keys are removed (paths are now hardcoded).

## Acceptance Criteria

- `tasks: <value>` is replaced with `tasks.backend: <value>`
- `task.list_limit` (if present) is moved to `tasks.list_limit`
- `task.analyse_limit` (if present) is moved to `tasks.analyse_limit`
- `recent_memory_days: <N>` is replaced with `memory.review_days: <N>`
- `memory:` key is removed
- `specs:` key is removed
- `groove-version:` is updated to `0.14.0`
- All other keys (`groovebook:`, `git.*`) are preserved unchanged

## Steps

1. Read `.groove/index.md` frontmatter
2. Extract current values:
   - `tasks:` → save as `backend` value (default: `beans`)
   - `task.list_limit` → save (default: `15`)
   - `task.analyse_limit` → save (default: `30`)
   - `recent_memory_days:` → save as `review_days` (default: `5`)
   - `groovebook:` → preserve
   - `git.*` → preserve
3. Remove old keys: `tasks:` (top-level scalar), `task:` (nested block), `memory:`, `specs:`, `recent_memory_days:`
4. Write new structure:
   ```yaml
   groove-version: 0.14.0
   groovebook: <preserved>
   tasks:
     backend: <backend>
     list_limit: <list_limit>
     analyse_limit: <analyse_limit>
   memory:
     review_days: <review_days>
   git:
     memory: <preserved>
     tasks: <preserved>
     hooks: <preserved>
   ```
5. Verify the rewritten frontmatter parses correctly

## Constraints

- Idempotent: if `tasks.backend` already exists, skip
- Preserve any unknown keys (future-proofing)
- Do not modify file content below the frontmatter closing `---`
