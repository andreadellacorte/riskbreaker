# Migration: 0.11.7 → 0.12.0

## What changed

v0.12.0 introduces:

- `<memory>/learned/` — long-term workflow insight directory
- `specs:` config key — required; specs directory (default: `<memory>/specs/`). Always set; if absent, config is broken.
- `groovebook:` config key — shared learning commons repo (default: `andreadellacorte/groovebook`)

## Outcome

Directory is created if absent. New config keys are added to `.groove/index.md` if missing. Existing content is not touched (idempotent).

## Steps

Read `memory:` from `.groove/index.md` (default: `.groove/memory/`). Let `<memory>` be that path.

### 1. Create `<memory>/learned/` directory

```bash
mkdir -p <memory>/learned
```

Report: `✓ <memory>/learned/ — created` or `✓ <memory>/learned/ — already present`

### 2. Add `specs:` to `.groove/index.md`

If `specs:` key is not present in the frontmatter, add it after the `recent_memory_days:` line. Value must be `<memory>/specs/` (e.g. if `memory:` is `.groove/memory/`, use `specs: .groove/memory/specs/`).

Report: `✓ specs: added to index.md` or `✓ specs: already present`

### 3. Add `groovebook:` to `.groove/index.md`

If `groovebook:` key is not present in the frontmatter, add it after the `specs:` line:

```yaml
groovebook: andreadellacorte/groovebook
```

Report: `✓ groovebook: added to index.md` or `✓ groovebook: already present`

## Constraints

- Idempotent — if directory or keys already exist, skip and report "already present"
- Do not overwrite existing values — only add keys that are absent
- Use the `memory:` value from `.groove/index.md`; default to `.groove/memory/` if key absent
- Preserve all existing frontmatter keys and formatting
