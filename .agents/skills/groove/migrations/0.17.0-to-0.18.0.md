# Migration: 0.17.0 → 0.18.0

## What changed

v0.18.0 fixes the hooks install skills and adds a version-check hook:

- **Claude Code**: `SessionStart` hook now runs the prime script directly instead of echoing a reminder. New `version-check` PostToolUse hook.
- **Cursor**: `sessionStart` hook now runs the prime script with `--json` instead of a separate `context-reprime.sh`. New `version-check` postToolUse hook. `context-reprime.sh` script removed.

## Outcome

Claude Code and Cursor hooks are reinstalled with the corrected configuration. Version-check hook scripts are written. Stale `context-reprime.sh` is removed from Cursor hooks directory.

## Steps

### 1. Update Claude Code hooks

Run `/groove-admin-claude-hooks` to reinstall all hooks with the corrected SessionStart command and new version-check hook.

Report: `✓ Claude Code hooks updated`

### 2. Update Cursor hooks (if applicable)

If `.cursor/` directory exists:

- Run `/groove-admin-cursor-hooks` to reinstall all hooks with the corrected sessionStart command and new version-check hook
- Remove `.groove/hooks/cursor/context-reprime.sh` if it exists (replaced by direct prime script call)

If `.cursor/` does not exist: skip silently.

Report: `✓ Cursor hooks updated` or `ℹ .cursor/ not present — skipping Cursor hooks`

## Constraints

- Idempotent — hook install skills check for duplicates before appending
- Do not create `.cursor/` if it does not exist
- Both hook skills will prompt the user for hook selection — default is all hooks
