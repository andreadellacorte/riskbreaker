# Migration: 0.8.8 → 0.8.9

## Outcome

Spec files live under `<memory>/specs/` instead of `<memory>/sessions/specs/`. Existing spec files are moved so `work spec` and `work audit` use a single location.

## Acceptance Criteria

- `<memory>/specs/` directory exists
- Every file and directory that was in `<memory>/sessions/specs/` has been moved to `<memory>/specs/` (same relative path)
- No files remain in `<memory>/sessions/specs/` after migration (directory may be empty or removed)
- Migration is idempotent — safe to re-run

## Constraints

- Read `memory:` from `.groove/index.md` frontmatter (default: `.groove/memory/`). Path is relative to git root.
- If `<memory>/sessions/specs/` does not exist, create `<memory>/specs/` if missing and exit
- When moving: for each item in `sessions/specs/`, if the target path under `specs/` already exists, skip that item; otherwise move (copy then delete, or rename)
- Preserve directory structure: e.g. `sessions/specs/foo/index.md` → `specs/foo/index.md`, `sessions/specs/bar.md` → `specs/bar.md`
- After moving all items, remove `<memory>/sessions/specs/` if empty (optional; leaving it empty is acceptable for idempotency)

## Steps

### Step 1 — Ensure `<memory>/specs/` exists

```bash
mkdir -p <memory>/specs
```

### Step 2 — Move contents of `sessions/specs/` to `specs/`

For each file or directory in `<memory>/sessions/specs/`:
- Target path = `<memory>/specs/<name>` (same name as in sessions/specs)
- If target already exists, skip
- Otherwise move the file or directory to `<memory>/specs/<name>`

(Use `mv` for atomic move, or copy then delete. For directories, move the whole directory so that e.g. `sessions/specs/topic/01-part.md` ends up as `specs/topic/01-part.md`.)

### Step 3 — Idempotency

If Step 2 is run again, targets already present under `specs/` are left unchanged; only items still in `sessions/specs/` are moved.
