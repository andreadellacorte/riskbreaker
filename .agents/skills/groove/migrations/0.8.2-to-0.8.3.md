# Migration: 0.8.2 → 0.8.3

## Outcome

`last-version-check:` is removed from `.groove/index.md`. `.groove/.cache/` exists with a `.gitkeep` and its contents are gitignored.

## Acceptance Criteria

- `last-version-check:` key is absent from `.groove/index.md` frontmatter
- `.groove/.cache/.gitkeep` exists
- `.groove/.gitignore` contains `.cache/*` and `!.cache/.gitkeep`
- If `last-version-check:` had a non-empty value, it is preserved in `.groove/.cache/last-version-check`

## Constraints

- This migration is idempotent — safe to re-run if interrupted

### Step 1 — Create `.groove/.cache/`

```bash
mkdir -p .groove/.cache
touch .groove/.cache/.gitkeep
```

If `.groove/.cache/.gitkeep` already exists, skip.

### Step 2 — Migrate `last-version-check:` value

Read `last-version-check:` from `.groove/index.md` frontmatter:
- If the value is non-empty and `.groove/.cache/last-version-check` does not exist, write the value as plain text to `.groove/.cache/last-version-check`
- Otherwise skip

### Step 3 — Remove `last-version-check:` from `.groove/index.md`

Remove the `last-version-check:` line from the frontmatter. If already absent, skip.

### Step 4 — Update `.groove/.gitignore`

Append the following lines to `.groove/.gitignore` if not already present:

```
# cache — always local
.cache/*
!.cache/.gitkeep
```
