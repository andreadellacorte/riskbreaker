# Migration: 0.18.3 → 0.18.4

## What changed

v0.18.4 fixes hook commands to use `$CLAUDE_PROJECT_DIR` instead of bare relative paths. Claude Code and Cursor hooks run with an unpredictable working directory, so relative paths like `bash .groove/hooks/...` resolve against wherever the shell was launched, not the project root.

## Outcome

Claude Code and Cursor hooks are reinstalled with `$CLAUDE_PROJECT_DIR`-prefixed commands so they resolve correctly regardless of working directory.

## Steps

### 1. Update Claude Code hooks

Run `/groove-admin-claude-hooks` to reinstall all hooks with `$CLAUDE_PROJECT_DIR`-prefixed commands.

Report: `✓ Claude Code hooks updated`

### 2. Update Cursor hooks (if applicable)

If `.cursor/` directory exists:

- Run `/groove-admin-cursor-hooks` to reinstall all hooks with `$CLAUDE_PROJECT_DIR`-prefixed commands.

If `.cursor/` does not exist: skip silently.

Report: `✓ Cursor hooks updated` or `ℹ .cursor/ not present — skipping Cursor hooks`

## Constraints

- Idempotent — hook install skills check for duplicates before appending
- Do not create `.cursor/` if it does not exist
- Both hook skills will prompt the user for hook selection — default is all hooks
