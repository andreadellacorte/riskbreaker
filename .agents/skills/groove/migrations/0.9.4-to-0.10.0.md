# Migration 0.9.4 → 0.10.0

Standalone `daily`, `work`, `task`, `memory` skills removed. All workflow commands now individual `groove-*` skills. No `.groove/index.md` config changes needed.

## Outcome

`AGENTS.md` task stub references updated from `/groove:task` to `/groove-utilities-task-*`.

## Acceptance Criteria

- `AGENTS.md` `<!-- groove:task:start -->` stub (if present) uses `/groove-utilities-task-*` skill names
- `.groove/index.md` `groove-version:` is `0.10.0` (set by groove update after this migration)

## Steps

1. Read `AGENTS.md` at git root
2. Locate the `<!-- groove:task:start -->` / `<!-- groove:task:end -->` section
3. If the section contains `/groove:task` (old format): replace stub content with:
   ```
   Task backend: beans — use `/groove-utilities-task-*` skills for all task management.
   Run `beans prime` to load the full beans CLI reference.
   ```
4. If the section is already updated or absent: skip silently

## Constraints

- Idempotent — safe to re-run; if already updated, do nothing
- Do not modify any other content in `AGENTS.md`
- Do not modify `.groove/index.md` — version bump is handled by `groove update` after this migration completes
