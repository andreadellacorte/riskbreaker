# Migration: 0.8.9 → 0.9.0

## Outcome

The obsolete `.agents/skills/skills` directory (removed in 0.7.0 when the skills wrapper skill was dropped) is deleted from existing installs so it no longer lingers.

## Acceptance Criteria

- If `.agents/skills/skills` exists at git root, it has been removed
- If it does not exist, migration is a no-op
- Migration is idempotent — safe to re-run

## Background

0.7.0 removed the `skills/skills` wrapper skill and all `skills *` routing from groove. New installs never received that folder, but existing installs were never cleaned up. This migration removes the leftover directory.

## Constraints

- All paths are relative to git root (`git rev-parse --show-toplevel`)
- Do not remove `.agents/skills/` itself — only the `skills` subdirectory (the former skill named "skills")
- If `.agents/skills/skills` is absent, skip

## Steps

### Step 1 — Remove `.agents/skills/skills` if present

From git root:

```bash
rm -rf .agents/skills/skills
```

If the path does not exist, the command is a no-op (idempotent).
