# Migration: 0.14.0 → 0.15.0

## What changed

v0.15.0 introduces:

- `context-reprime` hook — `SessionStart` hook for Claude Code that outputs a re-prime instruction after startup/compaction
- `groove-admin-cursor-hooks` skill — installs Cursor native hooks into `.cursor/hooks.json`

## Outcome

Claude Code hooks are updated with the new `context-reprime` hook (if already installed). Cursor hooks are installed if `.cursor/` exists. Users without existing hooks are prompted to install them.

## Steps

### 1. Update Claude Code hooks

Run `/groove-admin-claude-hooks` to install all hooks (including the new `context-reprime` hook).

- If `.claude/settings.json` exists and already contains groove hook entries: run the skill to add the new `context-reprime` hook alongside existing hooks
- If `.claude/settings.json` does not exist or has no groove hooks: run the skill to offer full hook installation

Report: `✓ Claude Code hooks updated` or `✓ Claude Code hooks installed`

### 2. Install Cursor hooks (if applicable)

If `.cursor/` directory exists:

- Run `/groove-admin-cursor-hooks` to install all Cursor hooks

If `.cursor/` does not exist: skip silently — Cursor is not in use.

Report: `✓ Cursor hooks installed` or `ℹ .cursor/ not present — skipping Cursor hooks`

## Constraints

- Idempotent — hook install skills check for duplicates before appending
- Do not create `.cursor/` if it does not exist — only install Cursor hooks for existing Cursor setups
- Both hook skills will prompt the user for hook selection — default is all hooks
