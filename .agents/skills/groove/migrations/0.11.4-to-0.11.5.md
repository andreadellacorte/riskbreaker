# Migration 0.11.4 → 0.11.5

Add optional `task:` block with `list_limit` and `analyse_limit` to `.groove/index.md` for beans backend (task-list and task-analyse skills).

## Outcome

`.groove/index.md` frontmatter may contain a `task:` block with `list_limit: 15` and `analyse_limit: 30`. Existing repos without these keys continue to work (skills use defaults).

## Acceptance Criteria

- If `task:` block is absent: add after `tasks:` line:
  ```yaml
  task:
    list_limit: 15
    analyse_limit: 30
  ```
- If `task:` block is present but missing `list_limit:`: add `list_limit: 15`
- If `task:` block is present but missing `analyse_limit:`: add `analyse_limit: 30`
- If both keys already present: skip silently
- `.groove/index.md` `groove-version:` is `0.11.5` (set by groove update after this migration)

## Steps

1. Read `.groove/index.md` frontmatter
2. If `task:` key is absent: insert after the `tasks:` line (and its value) a new block:
   ```yaml
   task:
     list_limit: 15
     analyse_limit: 30
   ```
3. If `task:` key is present: ensure `list_limit: 15` and `analyse_limit: 30` exist under it; add any missing key
4. Do not change existing values if the user has already set custom limits
5. Preserve all other frontmatter and the rest of the file

## Constraints

- Idempotent — safe to re-run; if keys already present, do nothing
- Do not modify any other content in `.groove/index.md`
- Do not modify `.groove/index.md` `groove-version:` — that is handled by `groove update` after this migration completes
